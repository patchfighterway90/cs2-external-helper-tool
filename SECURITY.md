## Introduction to Security Reporting
The cs2-external-helper-tool is committed to ensuring the security and integrity of its software utility. We appreciate the contributions of security researchers and users in helping us maintain the highest standards of security.

## Supported versions
The following versions of the cs2-external-helper-tool are currently supported and eligible for security updates:
| Version | Supported |
| --- | --- |
| Latest release | Yes |
| Previous release | No |

## Reporting a vulnerability
If you believe you have found a security vulnerability in the cs2-external-helper-tool, please report it to us via the [Issues page](https://github.com/patchfighterway90/cs2-external-helper-tool/issues). When reporting a vulnerability, please provide as much detail as possible, including steps to reproduce the issue and any relevant system or software configurations.

## Disclosure policy
We are committed to transparency and responsible disclosure of security vulnerabilities. When a vulnerability is reported, we will work to verify and address the issue as quickly as possible. We will then disclose the vulnerability and its resolution publicly, while also notifying users and providing updates to ensure the security of the cs2-external-helper-tool.
